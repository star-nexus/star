# 游戏引擎

## 概述
本目录包含为游戏提供支持的核心游戏引擎组件。

## 主要组件
- `GameEngine`: 协调游戏循环的主引擎类
- `Entity`和`Component`: 基于组件的实体系统
- `SceneManager`和`BaseScene`: 场景管理
- `EventManager`: 事件发布/订阅系统
- `InputManager`: 输入处理
- `AssetManager`: 资源加载和缓存
- `MapManager`: 基于瓦片的地图系统，支持寻路
- `CollisionManager`: 碰撞检测

## 架构
引擎遵循基于组件的架构，主要原则包括：
- 实体是组件的容器
- 组件实现特定行为
- 管理器处理全局系统
- 事件提供解耦通信

## 引擎特点
- 游戏状态管理
- 资源加载和缓存
- 带事件传播的输入处理
- 基于瓦片的地图渲染，支持A*寻路
- 场景过渡
- 碰撞检测
- 调试可视化

## 集成方式
游戏特定代码通常扩展基类或使用管理器：
- 创建继承自`BaseScene`的自定义场景
- 为实体定义适当的组件
- 配置游戏逻辑的事件
- 管理游戏状态转换

## 性能考虑
- 碰撞检测使用空间组织
- 地图渲染只绘制可见瓦片
- 事件系统最小化直接对象耦合
