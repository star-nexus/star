# 游戏管理器

## 概述
本目录包含处理全局游戏系统的管理器类。

## 主要管理器
- `GameStateManager`: 控制游戏状态转换和共享数据
- `CollisionManager`: 处理实体碰撞检测和响应
- `MapGenerator` (占位符): 将处理程序化地图生成

## 游戏状态管理
`GameStateManager`实现了一个状态机，包含以下状态：
- MENU: 主菜单显示
- PLAYING: 活动游戏
- PAUSED: 游戏暂停
- VICTORY/DEFEAT: 游戏结束状态

## 碰撞系统
`CollisionManager`提供：
- 实体间碰撞检测
- 实体与地图碰撞检测
- 选择性检测的碰撞组注册
- 碰撞回调和事件分发

## 集成
管理器通常在主游戏文件中初始化，并附加到引擎实例以进行全局访问。

## 最佳实践
- 尽可能使用事件系统进行通信而不是直接调用
- 保持管理器之间的清晰分离
- 优雅处理边缘情况（如与非活动实体的碰撞）
