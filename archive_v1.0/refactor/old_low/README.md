# 三国演义游戏

## 概述
这是一款基于三国演义的游戏，使用Python和Pygame实现。项目采用模块化架构设计，注重可维护性和可扩展性。

## 架构设计

### 分层架构
项目采用清晰的分层架构，各层次职责如下：

1. **核心引擎层** (`/engine`)
   - 提供游戏运行的底层支持
   - 实现核心循环、渲染和更新逻辑
   - 管理资源、事件分发和输入处理

2. **管理器层** (`/managers`)
   - 处理跨场景的全局状态
   - 实现游戏状态转换
   - 处理碰撞检测等全局系统

3. **场景层** (`/scenes`)
   - 实现不同的游戏状态和画面
   - 控制游戏流程和逻辑
   - 场景之间保持相互独立

4. **UI层** (`/ui`)
   - 提供用户界面组件
   - 实现交互元素和信息显示
   - 使用层次结构组织界面元素

5. **工厂层** (`/factories`)
   - 封装对象创建逻辑
   - 统一管理实体的实例化
   - 确保一致的组件配置

6. **内容生成层** (`/generators`)
   - 提供程序化内容生成功能
   - 动态创建游戏资源和关卡

### 设计模式应用

1. **组件模式**
   - 实体由多个组件组成 (Entity-Component)
   - 实现了组件的组合而非继承
   - 每个组件负责单一功能 (如渲染、碰撞、动画)

2. **控制器模式**
   - 游戏场景使用多个控制器分解复杂逻辑
   - 每个控制器负责特定方面 (如玩家控制、敌人AI、UI)
   - 控制器之间通过场景和事件系统通信

3. **工厂模式**
   - 使用EntityFactory创建和配置游戏实体
   - 封装复杂的实体创建过程
   - 确保组件配置的一致性

4. **状态模式**
   - GameStateManager实现游戏状态机
   - 管理游戏不同状态间的转换
   - 处理状态相关的数据和事件

5. **观察者模式**
   - 通过EventManager实现事件发布/订阅系统
   - 组件和系统之间通过事件进行解耦通信
   - 支持动态添加和移除事件监听器

6. **场景管理**
   - 使用SceneManager进行场景切换和管理
   - 场景封装特定游戏状态下的环境和逻辑
   - 场景之间通过GameStateManager传递数据

### 核心系统详解

1. **实体-组件系统**
   - Entity：游戏对象的容器
   - Component：提供特定功能的组件
   - 支持动态添加和移除组件
   - 实现了关注点分离

2. **事件系统**
   - 基于发布/订阅模式
   - 支持不同系统间的解耦通信
   - 处理用户输入、碰撞等事件

3. **渲染系统**
   - 分层渲染（地图、实体、UI）
   - 支持相机和视口管理
   - 优化仅渲染可见区域

4. **资源管理系统**
   - 集中加载和缓存游戏资源
   - 支持图像、音频、字体等资源类型
   - 提供资源的延迟加载和释放

5. **地图系统**
   - 基于瓦片的地图实现
   - 支持A*寻路算法
   - 提供碰撞检测和区域管理

## 项目结构
- `/engine`: 核心游戏引擎组件
- `/factories`: 实体创建的工厂类
- `/generators`: 程序化内容生成（开发中）
- `/managers`: 全局游戏系统管理
- `/scenes`: 不同游戏状态的场景
  - `/game_scene`: 主游戏场景控制器
- `/ui`: 用户界面框架

## 主要特点
- 基于组件的实体系统
- 控制器分离的游戏场景架构
- 瓦片地图系统，支持A*寻路
- 事件驱动通信模型
- 分层UI元素框架
- 状态机管理游戏流程

## 开发指南
1. **扩展实体**
   - 在EntityFactory中添加新的实体创建方法
   - 为实体配置适当的组件
   - 注册任何必要的碰撞组

2. **添加新场景**
   - 创建继承自BaseScene的新场景类
   - 实现必要的生命周期方法
   - 在SceneManager中注册场景

3. **创建UI组件**
   - 可以扩展现有UI元素或创建新类型
   - 通过UIManager注册和管理UI元素
   - 确保正确处理事件传播

4. **调试工具**
   - F1: 切换调试模式
   - F2: 切换FPS显示
   - F3: 切换玩家调试信息
