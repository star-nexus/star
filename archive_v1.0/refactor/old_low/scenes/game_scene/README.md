# 游戏场景

## 概述
本目录包含使用基于控制器架构的主游戏场景实现。

## 主要控制器
- `BaseController`: 所有控制器的抽象基类
- `PlayerController`: 管理玩家实体移动和输入
- `EnemyController`: 处理敌人行为和AI
- `MapHandler`: 管理游戏地图生成和寻路
- `UIController`: 管理游戏内UI元素
- `InputHandler`: 处理和路由游戏输入事件
- `GameScene`: 协调所有控制器的主场景类

## 架构
游戏场景使用模块化的控制器设计，每个控制器负责游戏的特定方面。这促进了关注点分离和可维护代码。

## 控制器生命周期
1. Initialize: 设置所需资源
2. Update: 每帧运行游戏逻辑
3. Cleanup: 释放不再需要的资源

## 通信方式
控制器通过以下方式通信：
- 从主场景直接引用控制器
- 存储在游戏状态管理器中的游戏数据
- 解耦交互的事件系统

## 最佳实践
- 以正确的顺序初始化控制器以处理依赖关系
- 使用调试模式辅助开发
- 在清理方法中正确释放资源
