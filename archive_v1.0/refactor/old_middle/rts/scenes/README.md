# 场景模块 (Scenes)

场景模块包含RTS游戏的各个场景定义，负责管理游戏的不同状态和界面呈现。

## 模块概述

场景系统管理游戏的各个状态转换和画面呈现，将游戏逻辑组织为相互独立但连贯的单元。每个场景都封装了特定游戏状态下的UI元素、输入处理和渲染逻辑。

## 主要场景

### 菜单场景 (RTSMenuScene)

游戏的主菜单界面：
- 提供开始游戏、设置和退出选项
- 简洁的游戏标题和界面元素
- 处理菜单导航和选择

### 游戏场景 (RTSGameScene)

游戏的主要游戏界面，包含完整的RTS游戏体验：
- 初始化和管理游戏世界、单位、建筑和资源
- 集成地图渲染和操作
- 处理用户输入和游戏命令
- 管理战斗和胜利条件

## 场景组件

游戏场景采用组件化设计，将功能拆分为多个管理器：

### UI管理器 (RTSUIManager)

管理游戏中的所有UI元素：
- 资源显示面板
- 小地图
- 实体信息面板
- 阵营和资源信息文本

### 输入处理器 (RTSInputHandler)

处理所有用户输入：
- 鼠标点击和拖拽操作
- 键盘命令
- 单位选择和命令
- 地图导航

### 地图管理器 (RTSMapManager)

管理游戏地图：
- 地图生成和加载
- 地图渲染和视图控制
- 提供寻路功能
- 坐标转换（屏幕与地图坐标）

### 实体管理器 (RTSEntityManager)

管理游戏中的所有实体：
- 资源节点管理
- 单位和建筑渲染
- 实体移除处理

### 战斗管理器 (RTSCombatManager)

处理游戏中的战斗效果：
- 处理攻击和伤害视觉效果
- 管理死亡和战斗事件

## 场景生命周期

场景的生命周期管理借助以下方法：

1. **load()**: 场景加载时调用，初始化UI和游戏元素
2. **unload()**: 场景卸载时调用，清理资源
3. **update(delta_time)**: 每帧更新场景状态和逻辑
4. **render()**: 渲染场景到屏幕
5. **process_event(event)**: 处理用户输入事件

## 场景转换

场景间的转换通过SceneManager实现：
- 从菜单场景到游戏场景
- 从游戏场景返回到菜单场景
- 胜利/失败状态的处理
