# 事件系统示例

这个示例展示了如何使用事件系统实现系统间的解耦通信。

## 功能特点

- **事件驱动架构**: 系统间通过事件通信，而非直接调用
- **发布-订阅模式**: 系统可以发布事件，其他系统可以订阅感兴趣的事件
- **松耦合设计**: 系统不需要直接了解其他系统的存在
- **事件链反应**: 一个事件可以触发多个系统的响应

## 包含的组件

1. **NameComponent** - 单位名称
2. **HealthComponent** - 生命值和相关方法
3. **WeaponComponent** - 武器信息（名称、伤害、命中率）
4. **ExperienceComponent** - 经验和等级系统

## 包含的系统

1. **CombatSystem** - 战斗系统

   - 随机选择单位进行攻击
   - 发布攻击、伤害、死亡等事件
   - 处理命中判定和伤害计算

2. **ExperienceSystem** - 经验系统

   - 订阅死亡事件，给击杀者经验
   - 处理升级逻辑
   - 发布升级事件

3. **HealingSystem** - 治疗系统

   - 定期治疗受伤单位
   - 发布治疗事件

4. **LoggingSystem** - 日志系统

   - 订阅所有战斗相关事件
   - 实时输出战斗日志
   - 展示事件的传播过程

5. **StatusDisplaySystem** - 状态显示系统
   - 定期显示所有单位的详细状态

## 事件类型

- `UNIT_ATTACK` - 单位攻击事件
- `UNIT_TAKE_DAMAGE` - 单位受伤事件
- `UNIT_DEATH` - 单位死亡事件
- `UNIT_LEVEL_UP` - 单位升级事件
- `UNIT_HEAL` - 单位治疗事件
- `COMBAT_LOG` - 战斗日志事件

## 运行示例

```bash
cd examples/event_demo
python main.py
```

## 预期输出

程序会创建几个三国武将进行战斗：

- **实时战斗日志**: 攻击、伤害、死亡、升级等事件
- **自动治疗**: 受伤单位会被定期治疗
- **经验系统**: 击杀敌人获得经验和升级
- **状态总览**: 定期显示所有单位的完整状态

## 事件流程示例

```
CombatSystem 发布攻击事件 → LoggingSystem 记录攻击日志
CombatSystem 发布伤害事件 → LoggingSystem 记录伤害信息
CombatSystem 发布死亡事件 → ExperienceSystem 给击杀者经验 → 可能触发升级事件
HealingSystem 发布治疗事件 → LoggingSystem 记录治疗信息
```

## 学习要点

1. **事件设计**: 如何设计合适的事件类型和数据结构
2. **系统解耦**: 通过事件实现系统间的松耦合
3. **事件订阅**: 如何正确订阅和处理事件
4. **事件链**: 一个事件如何触发其他事件形成事件链
5. **调试技巧**: 通过日志系统追踪事件流程
