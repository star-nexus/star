## 核心架构组件

### 1. 实体-组件-系统 (ECS)

游戏使用ECS架构来组织游戏对象和逻辑：

- **实体**：表示游戏对象（玩家、敌人、障碍物）的简单标识符
- **组件**：定义对象属性（位置、速度、碰撞器等）的纯数据容器
- **系统**：处理具有特定组件组合的实体的逻辑模块
- **世界**：管理所有实体、组件和系统的中央容器

这种方法将数据与行为分离，能够更好地组织代码和优化性能。

### 2. 管理子系统

多个专门的管理器处理游戏的不同方面：

- **事件管理器**：实现发布-订阅模式进行游戏范围的通信
- **输入管理器**：处理用户输入并生成适当的事件
- **渲染管理器**：处理基于层的绘图操作
- **场景管理器**：管理场景转换和生命周期，具有平滑的淡入淡出效果
- **资源管理器**：加载和缓存游戏资源（图像、字体）
- **音频管理器**：控制音效和背景音乐
- **UI管理器**：管理用户界面元素及其交互

### 3. UI框架

基于以下组件构建的分层UI系统：

- **UI元素**：具有通用功能的基类
  - **按钮**：交互式可点击元素
  - **面板**：容器UI元素
  - **标签**：文本显示元素

UI元素支持嵌套、事件处理和视觉定制。

### 4. 场景管理

游戏流程组织成不同的场景：

- **菜单场景**：具有游戏开始选项的标题屏幕
- **游戏场景**：具有实体和游戏逻辑的主要游戏区域
- **游戏结束场景**：胜利或失败屏幕，具有重玩选项

每个场景都有自己的生命周期方法（进入、更新、退出）并管理其特定的实体和UI。

### 5. 游戏特定组件

游戏对象由以下组件组成：

- **位置**：存储x、y坐标
- **速度**：定义移动方向和速度
- **碰撞器**：处理碰撞检测
- **可渲染**：视觉外观属性
- **玩家**：标记实体为可由玩家控制
- **敌人**：定义敌人行为属性
- **障碍物**：标记实体为静态障碍物

### 6. 游戏特定系统

游戏逻辑在以下系统中实现：

- **移动系统**：根据速度更新实体位置
- **玩家控制系统**：处理玩家输入以控制移动
- **敌人AI系统**：控制敌人向玩家移动
- **碰撞系统**：检测并响应实体碰撞
- **发光系统**：处理碰撞发生时的视觉效果
- **渲染系统**：将所有可渲染实体绘制到屏幕上

## 数据流

1. 引擎运行主游戏循环，管理时间并协调子系统
2. 输入事件由输入管理器捕获并转换为游戏事件
3. 事件管理器将事件分发给感兴趣的系统和管理器
4. 活动系统处理具有匹配组件的实体
5. 当前场景管理游戏状态和实体生命周期
6. 渲染管理器将最终场景绘制到屏幕上

## 初始化流程

1. 引擎创建子系统和世界
2. 游戏特定系统添加到世界中
3. 游戏场景注册到场景管理器中
4. 激活初始场景（菜单场景）
5. 主循环开始，处理输入、更新系统和渲染

该架构为游戏开发提供了一个灵活的基础，具有明确的关注点分离，使代码更易于维护和扩展。

## 关键设计模式

1. **实体-组件-系统**：核心架构模式，将数据与行为分离
2. **观察者模式**：事件管理器使用的松耦合系统
3. **单例模式**：用于需要全局访问的管理类
4. **状态模式**：通过场景系统实现的游戏状态管理
5. **命令模式**：用于UI交互处理程序
