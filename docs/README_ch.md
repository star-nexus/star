# 演示游戏

一个基于实体-组件-系统架构构建的简单游戏，用于展示游戏开发模式和实践。

## 概述

这个演示游戏实现了一个基本的追逐机制，玩家（蓝色圆圈）可能被敌人（红色圆圈）追逐。玩家需要获得足够的速度并与敌人碰撞以获胜。灰色障碍物提供了带有反弹效果的碰撞机制。

## 架构

该游戏使用ECS（实体-组件-系统）架构和全面的引擎框架构建。详细的架构概述请参见[Arch.md](Arch_ch.md)。

## 如何运行

1. 确保您安装了Python 3.12+和Pygame
   ```
   pip install pygame
   ```

2. 运行主脚本
   ```
   python main.py
   ```

## 游戏控制

- **菜单**：按空格键开始游戏
- **游戏**：
  - 方向键控制玩家移动
  - 积累足够的速度与敌人碰撞以击败它
- **游戏结束**：按R键返回菜单

## 关键组件

### ECS框架

- **实体**：简单的标识符（玩家、敌人、障碍物）
- **组件**：数据容器（位置、速度、碰撞体等）
- **系统**：逻辑模块（移动系统、碰撞系统等）

### 管理器子系统

- **事件管理器**：处理系统间的通信
- **输入管理器**：处理玩家输入
- **渲染管理器**：处理绘图操作
- **场景管理器**：控制游戏流程和场景转换
- **资源管理器**：加载和管理游戏资源
- **音频管理器**：处理音效和音乐

### 游戏场景

- **菜单场景**：标题屏幕
- **游戏场景**：主要游戏区域
- **游戏结束场景**：胜利或失败屏幕

## 游戏逻辑

1. 玩家需要在障碍物周围导航
2. 敌人不断追随玩家
3. 玩家在高速移动时与敌人碰撞即可获胜
4. 障碍物使实体在碰撞时反弹
5. 碰撞触发视觉发光效果

## 实现细节

1. **物理**：简单的圆形碰撞检测和基本的反弹物理
2. **AI**：敌人使用向量归一化追随玩家
3. **UI**：基本的文本渲染用于菜单和游戏状态
4. **场景转换**：游戏场景之间的平滑淡入淡出效果
5. **事件系统**：发布-订阅模式的游戏事件

## 扩展游戏

这个演示提供了一个可以扩展的基础：

1. 添加更多具有不同行为的敌人类型
2. 实现能量提升和特殊能力
3. 创建具有不同障碍布局的额外关卡
4. 添加计分和进度系统
5. 使用粒子效果和动画增强视觉效果
